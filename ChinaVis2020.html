ChinaVis2020参会总结

行业趋势

应用层
  1. 越来越多领域的科研团队利用可视化技术，寻找或验证规律、数据特点及异常点，以及对外交流中。例如：VIS for AI，用可视化表达可解释性。
  2. AI for VIS 方面中，根据数据特性自动生成可视化图表是比较多研究的课题，目前集中在算法的优化。另有利用AI配色或生成特定图表的研究，如 timeline，以及增强词云文本关系可视化的研究，例如 dancingword。
  3. 对于给艺术背景学生使用的可视化工具，还有很多缺陷，比如数据采集难、样式可调太少、代码编辑门槛高等等。
  4. 商业应用中多为数字大屏，现有的可视化工具样式不足，需要更多自定义功能。目前主流的开发平台是 Unity 3D 瓦片和 Web 3D 瓦片。
  5. 生物方面的可视化应用，主要集中在还原生理模型、验证信息传递路径、表现刺激-反应。
表现层
  1. 结合叙事学设计可视化作品，通过选择意象，融合动态数据变换，传递情感，进而加深观众对数据的感知。
  2. 大数据可视化的作品多为海报和交互网站，但这些传播范围都低，数据可视化视频易于传播、传播范围广，有更大的商业价值。
  3. 数据可视化还可以结合雕塑艺术、音乐等领域，完成一幅立体作品。每一个维度的数据交织在一起，多样而统一，表现力、感染力都非常震撼。

和自己的关联

  1. 情绪建模维度复杂，可以借用可视化手段帮助呈现情绪的多维状态。
  2. 情绪受时间、空间、行为、交际影响大，建模时可以借助可视化手段发现规律。
  3. 除了情绪，人的衣着、打扮、相貌都可以被数据化，但需要考虑怎么建模和 benchmark。
  4. 映射库除了参考数字媒体的意象、情绪外还可以有很多类型的表现形式，它甚至可以是中立的，只要越是利用到多维数据，越是能做到多样统一，映射出来的表现层就越有深度、越值得品味、越能打动人心。

主会场

Problem-Driven Vis

  1. 可视化四个角度的关注点：使用者、数据、目标、呈现

  2. 每个层级可能遇到的问题：搞错需求、抓错信息、低效可视、算法太慢

  3. 解决方案：从“技术引领”向“问题引领”转变设计思路

  4. “A design study is a project in which visualization researchers analyze a specific real-world problem faced by domain experts, design a visualization system that supports solving this problem, validate the design, and reflect about lessons learned in order to refine visualization design guidelines.”

  6. 可视化要点

Day3

可视化与艺术的跨界融合
  1. 视频类数据可视化最适合传播，容易出爆款，交互网页和海报都不够。
  2. 技术希望艺术提出需求：给艺术学生的数据采集（西班牙squech积木式）、数据清洗（不够简单）、可视化处理（自由度不够）。建议用D3。
可视化艺术项目 Translating Weather Data into Sculpture and Music

  1. 翻译成乐谱，多个维度的数据转变成多个声部的交织，形成乐章。这个过程是怎么做到的？

  2. 从环境中捕捉到多维数据，将多维数据转变成一个作品下的多为输入，并且产出的结果是非常多样而统一的。

  3. 相比国内的可视化作品，简直是降维打击……国内的作品像是二维作品，线条并排排列，但是她的作品是三维作品，线条交叉，建构立体感知。

  4. 她的作品是中立地表现数据，如何interpret，则多留给观众。

  5. 比较本次艺术项目来看，国内的可视化作品有以下特点：

    1. 多为数字媒体艺术、数字媒体技术结合的作品，媒介是多媒体。
    2. 维度单一，但艺术性强。
    3. 多个单一维度排列而成，维度和维度互相和谐呼应，但是彼此之间缺乏深度联系。
    4. 带有倾向性、情绪地解释数据。

  6. 中国人探索技术喜欢口嗨一堆大道理、指导纲领和思想，但落实到地有成果的很少。西方人做的东西比较能看到 step by step 的过程和结果。

虚拟重构兵马俑和非常秦腔
  1. 文物数据属于国家资源，不可能在外流通。
  2. 自己买点仿品砸、测量。
  3. 自己先做好了，再和组织谈合作。

ChinaVis2020参会总结

行业趋势

应用层
  1. 越来越多领域的科研团队利用可视化技术，寻找或验证规律、数据特点及异常点，以及对外交流中。例如：VIS for AI，用可视化表达可解释性。
  2. AI for VIS 方面中，根据数据特性自动生成可视化图表是比较多研究的课题，目前集中在算法的优化。另有利用AI配色或生成特定图表的研究，如 timeline，以及增强词云文本关系可视化的研究，例如 dancingword。
  3. 对于给艺术背景学生使用的可视化工具，还有很多缺陷，比如数据采集难、样式可调太少、代码编辑门槛高等等。
  4. 商业应用中多为数字大屏，现有的可视化工具样式不足，需要更多自定义功能。目前主流的开发平台是 Unity 3D 瓦片和 Web 3D 瓦片。
  5. 生物方面的可视化应用，主要集中在还原生理模型、验证信息传递路径、表现刺激-反应。
表现层
  1. 结合叙事学设计可视化作品，通过选择意象，融合动态数据变换，传递情感，进而加深观众对数据的感知。
  2. 大数据可视化的作品多为海报和交互网站，但这些传播范围都低，数据可视化视频易于传播、传播范围广，有更大的商业价值。
  3. 数据可视化还可以结合雕塑艺术、音乐等领域，完成一幅立体作品。每一个维度的数据交织在一起，多样而统一，表现力、感染力都非常震撼。

和自己的关联

  1. 情绪建模维度复杂，可以借用可视化手段帮助呈现情绪的多维状态。
  2. 情绪受时间、空间、行为、交际影响大，建模时可以借助可视化手段发现规律。
  3. 除了情绪，人的衣着、打扮、相貌都可以被数据化，但需要考虑怎么建模和 benchmark。
  4. 映射库除了参考数字媒体的意象、情绪外还可以有很多类型的表现形式，它甚至可以是中立的，只要越是利用到多维数据，越是能做到多样统一，映射出来的表现层就越有深度、越值得品味、越能打动人心。

可视化与艺术的跨界融合
  1. 视频类数据可视化最适合传播，容易出爆款，交互网页和海报都不够。
  2. 技术希望艺术提出需求：给艺术学生的数据采集（西班牙squech积木式）、数据清洗（不够简单）、可视化处理（自由度不够）。建议用D3。
可视化艺术项目 Translating Weather Data into Sculpture and Music

  1. 翻译成乐谱，多个维度的数据转变成多个声部的交织，形成乐章。这个过程是怎么做到的？

  2. 从环境中捕捉到多维数据，将多维数据转变成一个作品下的多为输入，并且产出的结果是非常多样而统一的。

  3. 相比国内的可视化作品，简直是降维打击……国内的作品像是二维作品，线条并排排列，但是她的作品是三维作品，线条交叉，建构立体感知。

  4. 她的作品是中立地表现数据，如何interpret，则多留给观众。

  5. 比较本次艺术项目来看，国内的可视化作品有以下特点：

    1. 多为数字媒体艺术、数字媒体技术结合的作品，媒介是多媒体。
    2. 维度单一，但艺术性强。
    3. 多个单一维度排列而成，维度和维度互相和谐呼应，但是彼此之间缺乏深度联系。
    4. 带有倾向性、情绪地解释数据。

  6. 中国人探索技术喜欢口嗨一堆大道理、指导纲领和思想，但落实到地有成果的很少。西方人做的东西比较能看到 step by step 的过程和结果。
